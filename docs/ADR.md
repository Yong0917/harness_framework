# Architecture Decision Records

## 철학

이식성과 단순성 최우선. 외부 의존성 없이 Python 표준 라이브러리만 사용. 복잡한 추상화보다 직관적인 단일 파일 구현을 선택.

---

### ADR-001: execute.py 단일 파일 유지

**결정**: `execute.py`를 헬퍼 모듈로 분리하지 않는다.

**이유**: `python3 scripts/execute.py`로 설치 없이 즉시 실행 가능해야 한다. 파일이 하나면 복사해서 어디서든 쓸 수 있다.

**트레이드오프**: 파일이 길어질 수 있지만(현재 ~420줄) 이식성이 최우선이다.

---

### ADR-002: Claude CLI 직접 호출 (subprocess)

**결정**: Anthropic Python SDK 대신 `claude -p` CLI를 `subprocess.run`으로 호출한다.

**이유**: Claude Code의 파일 편집/터미널 툴을 활용해야 한다. SDK는 텍스트 생성만 지원하고 툴 실행 권한이 없다.

**트레이드오프**: 반환값이 JSON stdout이므로 파싱이 필요하다. Claude CLI가 설치돼 있어야 한다.

---

### ADR-003: JSON 상태 파일로 실행 추적

**결정**: 실행 상태를 인메모리가 아닌 `phases/{task-name}/index.json`에 저장한다.

**이유**: 프로세스가 중단돼도 마지막 상태를 보존하고 재시작할 수 있어야 한다. 상태가 파일에 있으면 사람이 직접 편집해서 재시도/건너뛰기도 가능하다.

**트레이드오프**: 매 step마다 파일 I/O가 발생하지만 성능보다 안정성이 중요하다.

---

### ADR-004: 재시도 최대 3회 고정

**결정**: `MAX_RETRIES = 3`으로 고정. CLI 인자로 노출하지 않는다.

**이유**: 같은 프롬프트로 3번 이상 실패하면 step 설계 자체에 문제가 있을 가능성이 높다. 무한 재시도는 비용 낭비다.

**트레이드오프**: 특수한 경우 재시도 수를 조정할 수 없지만, 대부분의 경우 3회로 충분하다.
